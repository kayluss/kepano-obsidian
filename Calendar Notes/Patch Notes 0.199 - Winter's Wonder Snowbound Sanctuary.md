### Lunar Path Improvements

Additionally, when you log in you‚Äôll notice a few changes to Lunar Paths. Starting in Patch 0.199, you‚Äôll be able to **earn Lunar XP simply by earning Skill XP** doing all of the activities you already love in Palia. This means those of you who want to grind out Path rewards separate from Dailies and Bundles will have that option - but things like Dailies and Bundles will still be the most efficient sources of Lunar XP.

With the addition of this new system, a percentage of your Skill XP earned will additionally be awarded as Lunar XP, applying directly to your active Path passively. You‚Äôll also be able to track your progress directly on the HUD!

But we don‚Äôt want players to feel like grinding Lunar Paths needs to be a second job - so we‚Äôre also introducing a bonus system to help players catch up who don‚Äôt have as much time to spend: **Rest XP!** Every new Palia day (each real life hour), you‚Äôll bank a certain amount of Rest XP which will multiply the Lunar XP from skills by 10x until it is used up. If you don‚Äôt use it, you can bank up to 24 hours of this bonus XP so that the next time you log in you can really catch up quickly. We think this will help make sure players of all commitment levels can earn rewards at a reasonable pace.

As part of adding these new sources of Lunar XP, we‚Äôre also embiggening all of the Lunar XP numbers - by 100x. For instance, Dailies and Bundles will now give 5,000 Lunar XP, and Path levels will take 10,000 Lunar XP to complete. All of your current progress with the Path of the Kitsuu will remain the same - just with these new values - so don‚Äôt be alarmed when you log in to find everything is bigger than you remember!

We‚Äôre still learning a lot about Lunar Paths, so please continue to provide feedback and expect further changes as time goes on. Right now, we‚Äôll be monitoring the new Skill XP source to make sure it‚Äôs fair and fun. We especially have our eyes on Furniture Making and how it contributes to Lunar XP, but for now it works the same as every other skill, so try it out before we make any changes in a future patch.

### Tint Tool Updates

![copyMode01](https://images.ctfassets.net/qyr5bjm559gz/7fxgf1msDeZm2SHhBCcoO5/c61894d05ad38dddfa313501fd0eee41/copyMode01.gif)

We‚Äôre expanding the tint system from Patch 0.198 with a brand-new feature: Copy Mode! Everything you could do in the previous tint update still works, but now you have a faster, easier way to apply colors across your decor.

There are two ways to enter Copy Mode: either select the new button in the Tint UI while in Placement Mode, or use a tint item from your hotbar to jump straight into Copy Mode.

## Top Bug Fixes

***Disclaimer:** Please note this is not a comprehensive list of all fixed issues.*

We put in a lot of work this cycle to clean things up across Palia. Tons of bugs were fixed, and we have listened to player feedback to make improvements in lots of areas to help everything run smoother and feel better to play.

- Fixed an issue where Pavers were incorrectly sized.
- Fixed an error regarding the Maji Market operational schedule.
- Resolved an issue where Weekly Wants would fail to reveal if the requirements were met before the player interacted with the "Discover" button.
- Fixed a visual flickering issue where daily quest cards would briefly disappear when navigating the Bulletin Board menu.
- Added the missing description for the Adaptive Trigger setting within the Controller Settings menu.
- Fixed an issue where the Brighteye Amulet and Tincture of Infected Perception failed to correctly boost Echo fish loot rates.
- Fixed a visual UI bug where Worms incorrectly displayed a placeholder or wrong icon
- Fixed an issue where firework sound effects failed to play while in the Fairgrounds.
- Resolved an issue where the Chapaa Chase minigame music would continue playing after the match concluded.
- Improved the responsiveness and navigation speed when viewing reward popups on the Bulletin Board.
- Resolved an issue where Kenli‚Äôs dialogue was not translated (unlocalized) during the "Bulletin Blitz" quest.
- Fixed a map issue where the Bulletin Board quest marker incorrectly displayed notifications for completed or past quests.
- Updated the Shipping Bin interface text from "Swap" to "Move to Inventory" for better clarity.
- Resolved an exploit where Muujins could be killed repeatedly if they were struck at the exact moment they jumped into a tree.
- Fixed a progression issue where the Bulletin Board Outro quest could be triggered incorrectly before all necessary requirements were met.
- Corrected a typo in the item description for the "Rummaged Goodies Bag" item.
- Corrected spelling errors in the description for the Angler‚Äôs Fortune item.
- Resolved a UI issue where item tooltips were appearing larger than intended on the Bulletin and Bundle Boards.
- Fixed a UI issue where the XP preview bar failed to update when only a portion of a sub-bundle was completed.

### Housing & Decor

- Fixed an issue where tint previews would not appear if the player accessed the menu while in Copy Mode.
- Resolved an issue preventing players from rotating furniture and decor while in the tint preview window.

### Accounts, Character Creation, Collections, and Wardrobe

- Fixed a logic error where the Diva trait effect applies without cosmetic traits applied.
- Fixed an issue where Kilima Coins were contributing toward the "Medal Earner" achievement.
- Resolved a delay where the Progress Bar would not update immediately after a player completed a daily quest.

### Premium Cosmetics

- Fixed some issues where transfer notifications were not exitable.
- Equipping a Boovar as a pet now properly displays the Boovar's icon in the character menu.
- Color picker will now show additional options for players during Character Creation
- Fix made for players getting yeeted off the screen at the end of Character Creation.
- The "Equip" option after purchasing an outfit is now fixed.
